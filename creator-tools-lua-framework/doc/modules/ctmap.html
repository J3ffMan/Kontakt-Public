Module ctmap

Creator Tools LUA Mapping Utilities File

Author: Native Instruments Written by: Yaron Eshkar

This file includes useful functions for usage in Creator Tools Lua scripts.

Simply include this line in any script (if running a script from another location that users this file, make sure to point to the correct path): local ctMap = require("CtMap")

Then you can simply call any function like: ctMap.test_function()

It is also possible of course to copy entire specific functions from this list directly into your script. In that case remove the CtMap part from the function name, and then simply call it normally like: test_function()

CtMap.create_mapping (sample_paths_table, sample_tokens_table, playback_mode, sample_name_location, signal_name_location, articulation_location, round_robin_location, root_detect, root_location, key_confine, low_key_location, high_key_location, vel_confine, low_vel_location, high_vel_location, set_loop, loop_xfade, default_root_value, default_low_key_value, default_high_key_value, default_low_vel_value, default_high_vel_value, verbose_mode, fix_tune)
Creates a sample map for Kontakt. Generic sample map function that takes many argument and applies the resulting map to the Kontakt mapping editor.

Parameters:

  • sample_paths_table table table containing the paths to the sample files.
  • sample_tokens_table table table containing the tokens contained in the name of each sample file.
  • playback_mode string sets the playback mode for the created groups. Can be one of:

    dfd

    sampler

    wavetable

    tone_machine

    time_machine

    time_machine_2

    time_machine_pro

    beat_machine

    s1200

    mp60

  • sample_name_location int the expected location of the sample name token in the sample_tokens_table.
  • signal_name_location int the expected location of the signal name token in the sample_tokens_table.
  • articulation_location int the expected location of the articulation token in the sample_tokens_table.
  • round_robin_location int the expected location of the round robin token in the sample_tokens_table.
  • root_detect bool if set to true, root will be detected using MIR. When set to false, root will be set manually by the relvent variables.
  • root_location int the expected location of the root note token in the sample_tokens_table.

    Set to 0 to use the default value, set to -1 to not set the parameter at all.

  • key_confine bool when set to true, the zone low key, high key will be the same as the root key.
  • low_key_location int the expected location of the low key token in the sample_tokens_table.

    Set to 0 to use the default value, set to -1 to not set the parameter at all.

  • high_key_location int the expected location of the high key token in the sample_tokens_table.

    Set to 0 to use the default value, set to -1 to not set the parameter at all.

  • vel_confine bool if set to true, low vel and high vel will be set to the default parameters.
  • low_vel_location int the expected location of the low vel token in the sample_tokens_table.

    Set to 0 to use the default value, set to -1 to not set the parameter at all.

  • high_vel_location int the expected location of the high vel token in the sample_tokens_table.

    Set to 0 to use the default value, set to -1 to not set the parameter at all.

  • set_loop int

    0: No loop.

    1: Turn on the first loop of the sample in mode 1.

    2: Turn on the first loop of the sample in mode 1 and set the loop parameters.

  • loop_xfade int anything greater than 0 sets the xfade to the given value.
  • default_root_value int the default root value to use if explicity told too or if an error is encountered parsing the token.
  • default_low_key_value int the default low key value to use if explicity told too or if an error is encountered parsing the token.
  • default_high_key_value int the default high key value to use if explicity told too or if an error is encountered parsing the token.
  • default_low_vel_value int the default low vel value to use if explicity told too or if an error is encountered parsing the token.
  • default_high_vel_value int the default high vel value to use if explicity told too or if an error is encountered parsing the token.
  • verbose_mode bool when true, prints the results of the mapping operation to console.
  • fix_tune bool When true MIR is used to fine tune each sample to the nearest note.

Returns:

    bool
