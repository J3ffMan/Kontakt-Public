Module ctmir

Creator Tools LUA MIR Utilities File

Author: Native Instruments Written by: Yaron Eshkar

This file includes useful functions for usage in Creator Tools Lua scripts.

Simply include this line in any script (if running a script from another location that users this file, make sure to point to the correct path): local ctMir = require("CtMir")

Then you can simply call any function like: ctMir.test_function()

It is also possible of course to copy entire specific functions from this list directly into your script. In that case remove the CtMir part from the function name, and then simply call it normally like: test_function()

CtMir.mir_detect (path, mir_type, batch, frame_size_in_seconds, hop_size_in_seconds)
Perform any of the MIR operations.

Parameters:

  • path string the path to the audio files. A directory for batch operation, a file for a single operation.
  • mir_type string the type of MIR operation to perform. Can be one of:

    sample

    drum

    instrument

    pitch

    peak

    rms

    loudness

  • batch bool when true performs the operation on a directory, otherwise on a single file.
  • frame_size_in_seconds float the frame size for RMS and loudness detection. Degfault to 0.4.
  • hop_size_in_seconds float the hop size for RMS and loudness detection. Defaults to 0.1.

Returns:

    return the operation result. Batch returns a table, single returns a variable.
CtMir.mir_test (perform_batch, perform_single, mir_path_batch, mir_path_single)
Run MIR detection on a folder or single file and print the results.

Parameters:

  • perform_batch bool when true performs the batch operation.
  • perform_single bool when true performs the single operation.
  • mir_path_batch string the directory path containing audio files for batch operation.
  • mir_path_single string the file path to an audio file for single file operation.

CtMir.type_to_tag (var, mode, mir_path_single)
Helper function for implementing MIR type tags to strings.

Parameters:

  • var table or enum. If called with a single MIR operation, var is a string and the mir_path_single argument must be present.
  • mode string the mode to pass to the next function. Can be one of:

    type

    drum

    instrument

  • mir_path_single string if the input type is not a table, path must be provided.

Returns:

    var returns either a string or a table depending on the input type.
