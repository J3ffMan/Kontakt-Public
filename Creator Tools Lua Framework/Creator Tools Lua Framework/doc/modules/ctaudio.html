Module ctaudio

Creator Tools LUA Audio Utilities File

Author: Native Instruments Written by: Yaron Eshkar

This file includes useful functions for usage in Creator Tools Lua scripts.

Simply include this line in any script (if running a script from another location that users this file, make sure to point to the correct path): local ctAudio = require("CtAudio") Then you can simply call any function like: ctAudio.test_function()

It is also possible of course to copy entire specific functions from this list directly into your script. In that case remove the CtAudio part from the function name, and then simply call it normally like: test_function()

CtAudio.kick_synth (output_file, sample_rate, length, start_freq, end_freq, bend_start, bend_end, bend_freq, bend_twice, fade_time, verbose_mode)
Kick Synth. kick audio sample generator.

Parameters:

  • output_file string output_file the desired path of the output file. Defaults to kick.wav in the directory where script was run.
  • sample_rate string the sample rate of the resulting files. Defaults to 44100.
  • length string the total length of the sample in seconds. Defaults to 0.5
  • start_freq string the starting frequency of the oscillation. Defaults to 100.
  • end_freq string the end frequency of the oscillation. Defaults to 40.
  • bend_start string the place in the sample from which to start the bend operation in % of overall sample length. Defaults to 0.
  • bend_end string the place in the sample where the bend operation will end in % of overall sample length. Defaults to 0.25.
  • bend_freq string the amount of pitch bend to apply over time from bend_start to bend_end, in cents. Defaults to -1200.
  • bend_twice bool when true, performs the bend operation twice.
  • fade_time string the volume fade out time, in % of overall sample length.
  • verbose_mode bool when true prints information to console.

Returns:

    bool

CtAudio.check_sample_set_integrity (path, verbose_mode, perform_rms_check, deviance_rms_factor, perform_loudness_check, deviance_loudness_factor, perform_peak_check, deviance_peak_factor, perform_rate_check, expected_sample_rate, perform_depth_check, expected_bit_depth, perform_loop_check, expected_loop, perform_file_size_check, deviance_file_size_factor, perform_file_names_check)
Check the integrity of a directory of samples using various comparison metrics.

Parameters:

  • path string the path to the directory of samples to check and compare.
  • verbose_mode bool when true prints information to console.
  • perform_rms_check bool when true, compare RMS values.
  • deviance_rms_factor number the accpetable deviance in RMS valus between the files.
  • perform_loudness_check bool when true, compare loudness values.
  • deviance_loudness_factor number the accpetable deviance in loudness valus between the files.
  • perform_peak_check bool when true, compare peak values.
  • deviance_peak_factor number the accpetable deviance in peak valus between the files.
  • perform_rate_check bool when true, compare sample rate values.
  • expected_sample_rate int the expected sample rate. Defaults to 44100.
  • perform_depth_check bool when true, compare bit depth values.
  • expected_bit_depth int the expected bit depth. Defaults to 24.
  • perform_loop_check bool when true, compare check if loop points exists.
  • expected_loop bool when true and performing a loop check, the check should pass if a loop exists.
  • perform_file_size_check bool when true, compare file size values.
  • deviance_file_size_factor int the acceptable different between the file sizes.
  • perform_file_names_check bool when true, compare file names, if duplicates exist the check fails.

Returns:

    bool returns true when all checks pass, false when any check fails.
